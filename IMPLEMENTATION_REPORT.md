# Implementation report

## Исправленные критические ошибки

- Fixture-источник отключён в production и валидатор запрещает включить его там.
- Identity v2 не зависит от изменяемых заголовка, времени и AI-текста.
- Неизвестная дата больше не подменяется текущим временем.
- Первичный снимок запоминается без рассылки архива; backfill включается только явно.

## Архитектурные изменения

- Добавлены модели конфигурации AI/Telegram, `SourceResult` и коды результата health.
- Унифицированы лимиты HTTP-ответов, типы контента и retry-after.

## Изменения источников

- Универсальный HTML-парсер сначала использует контейнери публикаций и отбрасывает
  навигацию, архивы и ссылки на другой домен.

## Дедупликация

- State хранит постоянный compact `identity_index`; очистка детальной истории не
  делает старую публикацию новой.

## Миграции состояния

- Реализованы последовательные миграции 4→5→6→7, backup перед миграцией,
  отказ от будущей версии и детерминированная функция merge состояния.

## Telegram

- Учитывается `retry_after`, сохраняется `message_id`, а лимит сообщений за запуск
  и отложенная отправка low-priority настраиваются в YAML.

## GitHub Models

- Введены дневной и запускной бюджет, резерв high-priority, TTL/LRU-ограничение
  кеша и статистика cache hits/skips. При ошибке доставка не блокируется.

## Использование бесплатных AI-лимитов

- Значения по умолчанию: 10 запросов/запуск, 100/день, резерв 3.
- Лимиты и отключение AI находятся в блоке `ai` файла `config/sources.yaml`.
- При rate limit запросы прекращаются для текущего запуска; используется fallback.

## GitHub Actions

- Code quality проверяет форматирование. Добавлены Dependabot и еженедельный
  smoke-test без Telegram, записи state или commit.

## Безопасность

- XML перед разбором ограничивается размером и отвергается при DOCTYPE/entity.
- Токены не добавляются в диагностику; HTTP-клиент не повторяет 403/404.

## Добавленные тесты

- Production fixture validation, identity v2, неизвестные даты, миграции/merge
  состояния и Telegram retry_after.

## Результаты проверок

- `python -m pytest --basetemp .pytest-tmp`: 68 passed.
- `ruff check .`: passed.
- `ruff format --check .`: passed.

## Ограничения

- Внешние сервисы остаются Telegram Bot API и опциональный GitHub Models.
  Их тарифы и лимиты могут меняться.
- Git/Telegram дают best-effort idempotency, а не математически строгую exactly-once доставку.

## Ручные действия владельца

- Проверить фактические лимиты GitHub Actions и GitHub Models для своего плана.
- Добавить Telegram secrets только в GitHub Actions Secrets.
